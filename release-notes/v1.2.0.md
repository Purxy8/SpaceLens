# SpaceLens 1.2.0

SpaceLens 1.2 makes storage totals easier to read, gives the app a friendlier visual refresh, and strengthens the scan-to-cleanup workflow from end to end.

## What changed

- Storage sizes now use familiar decimal `B`, `KB`, `MB`, `GB`, and `TB` labels throughout the interface. For clarity, `1 GB = 1,000,000,000 bytes`.
- The interface has refreshed metric cards, cleaner spacing and color, clearer scan/cache states, smoother table presentation, and a subtle activity animation while work is running.
- Large scans filter, sort, categorize, and refresh more efficiently, keeping the interface more responsive without allowing a column sort to reset the selected category.
- Sparse, compressed, and unusual files are handled more honestly: if Windows cannot report physical allocation, SpaceLens marks the value as an estimate instead of silently counting zero bytes.
- Scan traversal and completed-scan cache validation are stricter, including safer linked-directory handling, root/volume identity checks, and ordered cache writes.
- Recycle Bin cleanup validates that a selected file is still the same file recorded by the scan, and bulk removal no longer performs repeated full-list searches.
- Setup now enforces exact payload/product versions, rejects downgrades, preserves shortcut choices during upgrades, blocks interruption during the critical install phase, and restores prior files and registration if an upgrade fails.
- The release workflow now separates intermediate files from publishable assets and asks the packaged app itself to verify the signed manifest and exact installer before the six release files are promoted.

## Notes

Windows drive-used space remains authoritative. Indexed file totals can still be lower because NTFS metadata, restore points, reserved storage, inaccessible locations, and other filesystem overhead are not ordinary deletable files.

SpaceLens 1.2 uses a stricter saved-scan format. After updating from 1.1, run one fresh scan; completed scans will then load automatically on later launches as before.

The binaries are not yet Authenticode-signed, so Windows SmartScreen may show an unknown-publisher warning. SpaceLens update metadata remains independently protected by its pinned ECDSA verification key and exact installer hash.
