# SpaceLens 1.3.0

SpaceLens 1.3 makes full-drive scans substantially faster and makes the disk accounting easier to trust.

## What changed

- The scanner now reads Windows directory metadata in large native buffers instead of opening almost every file separately. It keeps physical allocation, timestamps, file identity, sparse/compressed handling, linked-directory safety, and hard-link deduplication while greatly improving throughput.
- The status bar shows live files per second, and scan results reach the interface in larger async-safe batches to avoid UI queue lag.
- File allocations that Windows could only estimate are now included and visibly marked. Earlier versions found these files but excluded their bytes from totals, which could make large Windows files look like unexplained missing disk space.
- The old “Unindexed / overhead” card is now split into clearer “Files accounted” and “System / protected” concepts. The latter is calculated from the same drive-used snapshot and represents Windows-managed or inaccessible storage rather than a promise of deletable files.
- Completed scans now save the drive-used snapshot, skipped-location count, and scan duration. Version 1.2 caches remain readable, but they show `RESCAN` for system/protected storage until one fresh scan records the new summary.
- Changing a filter or sort temporarily disables file actions, preventing a stale row from being recycled while the next background view is still being calculated.

## Verification

The release regression suite covers native scanner activation, throughput, sparse files, NTFS hard links, linked-directory loops, long paths, cancellation, legacy cache compatibility, composable filters/sorts, safe recycle validation, updater signature verification, and UI construction. The packaged application and installer are self-tested again during signed release creation.

## Notes

Windows drive-used space remains authoritative. “System / protected” can include restore points, System Volume Information, reserved storage, NTFS metadata, inaccessible recycle-bin data, and alternate streams. SpaceLens does not label that residual as ordinary deletable files.

The binaries are not yet Authenticode-signed, so Windows SmartScreen may show an unknown-publisher warning. SpaceLens update metadata remains independently protected by its pinned ECDSA verification key and exact installer hash.
