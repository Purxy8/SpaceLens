# SpaceLens 1.1.0

SpaceLens 1.1 is the first update-enabled release. Users upgrading from 1.0 must install this version manually once; later releases can then be discovered from inside the application.

## Added

- Manual **Check for updates** button in the application header
- Silent daily update checks that never block startup
- Strict semantic-version comparison
- ECDSA P-256 signed update manifests with a public verification key pinned inside SpaceLens
- Fixed GitHub repository, release tag, and installer asset validation
- Size-bounded, cancellable streaming downloads
- Exact byte-count, SHA-256, PE-header, and manifest-signature verification before installer launch
- Upgrade-aware Setup mode that waits for SpaceLens to close and preserves cache/preferences

## Security behavior

- Signature verification is mandatory and cannot be disabled by the manifest.
- The manifest cannot provide an arbitrary executable URL or command-line arguments.
- Same-version, downgrade, malformed, duplicate-field, oversized, unsigned, or modified manifests are rejected.
- Modified, truncated, oversized, or hash-mismatched installers are deleted and never launched.

The binaries remain unsigned with Authenticode, so Windows SmartScreen may still warn. The independently signed update manifest authenticates in-app update metadata and installer hashes; Authenticode signing remains recommended before broad public distribution.
