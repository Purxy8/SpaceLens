# SpaceLens 1.5.0

SpaceLens 1.5 fixes the Full access scan startup failure and substantially improves scan transport, memory use, interface responsiveness, and security validation.

## Fixed

- Full access scan no longer closes immediately after UAC on filtered Administrator accounts.
- Elevated startup failures now report a useful helper error/exit code while preserving previous results.
- Scan roots, aliases, and every traversed directory are resolved and identity-checked on Windows handles so junctions cannot escape the selected location.
- Cached root identity, directory-record validation, and bounded parsing prevent stale or hostile cache data from being trusted.
- Category/sort/filter state remains composable, and scan rate now measures actual scan work instead of the UAC wait.

## Faster and smoother

- Pooled 8,000-file elevated frames remove repeated record/frame copies and large-object allocation churn.
- Category codes eliminate millions of duplicate category strings across IPC and v6 caches.
- Classification, media detection, totals, rescans, view updates, and cache loading allocate less and perform less duplicate path work.
- Recycle operations run on a background STA worker so the interface stays responsive.

## Security hardening

- Cleanup revalidates final path, type, timestamps, safety level, and stable file identity immediately before each Recycle Bin-only operation.
- Directory rows, path escapes, changed identities, and ambiguous sensitive files fail closed.
- Update manifests and local state are read through strict size bounds, including chunked responses.
- Developer diagnostics are compiled out of production packages, and framework-dependent `dotnet.exe` builds cannot self-elevate.

## Download

- **Recommended:** `SpaceLens-Setup.exe` installs or upgrades SpaceLens and maintains Desktop/Start Menu shortcuts.
- **Portable:** `SpaceLens.exe` runs directly without installation.

The binaries are not currently Authenticode-signed, so Windows may show an unknown-publisher warning. Download only from the official GitHub Releases page and use the accompanying `.sha256` files for manual verification.
