# SpaceLens 1.6.1

SpaceLens 1.6.1 is a defense-in-depth security update for the application, elevated scan boundary, updater, and release supply chain. It also retains the performance and interface improvements introduced in 1.6.0.

## Important manual upgrade

The obsolete development private key was removed, and a new non-exportable, user-protected CNG key plus its matching public self-test fixture were enrolled for this release. Because SpaceLens 1.6.0 and earlier cannot securely authenticate this trust-anchor change, they must **not** install 1.6.1 solely through the old in-app update channel. For this one transition, download `SpaceLens-Setup.exe` manually from the official `Purxy8/SpaceLens` GitHub release page and compare its SHA-256 with the accompanying sidecar. After 1.6.1 is installed, later in-app updates can use the newly pinned key normally.

## Security hardening

- Build/self-test and offline signing are now separate, exact-hash-bound phases.
- The maintainer completed the one-shot user-session handoff, which created the preferred non-exportable, user-protected Windows CNG ECDSA P-256 key, exported only its public key, and transactionally installed a freshly signed self-test fixture.
- Exportable private PEM generation/signing is disabled; release signing accepts only the non-exportable, high-protection CNG key.
- The signer performs strict bounded parsing, P-256 enforcement, signing, public-key verification, and exact-output verification in one process.
- The offline finalizer never executes SpaceLens or Setup on the key-holding account; it parses and hashes their bytes and launches only an independently hashed, locked NativeAOT signer.
- Active GitHub Actions are immutable-SHA pinned and isolated from untrusted pull-request code. The unsafe single-runner SignPath design was retired; no usable signing workflow exists pending an independently audited split-job replacement.
- CI proves the production SpaceLens apphost ignores a harmless `DOTNET_STARTUP_HOOKS` fixture.
- Full access/NTFS refresh and all UAC-helper entry points are temporarily disabled pending a native broker; ordinary scanning, UI, cleanup, update, install, and uninstall remain unelevated and reject elevated operation. SpaceLens 1.6.1 never requests UAC—cancel and report any unexpected prompt.
- Preserved scan protocol parsing binds root identity, enforces cumulative IPC budgets, and applies case-sensitive canonical containment.
- Updater redirects use an exact HTTPS host allowlist, and installer bytes stay locked and rehashed through launch.
- Setup and uninstall use hardened identity-bound staging, ownership checks, and canonical path validation.
- Native and fallback scans now enforce bounded directory, pending-work, file-record, path-memory, cache, and file-type aggregation budgets; partial-limit results are never saved as complete.
- Managed fallback cancellation can no longer be swallowed as a skipped location, and unexpected fatal/runtime failures are no longer hidden by broad filesystem catches.
- Update checks time out after one minute while verified installer downloads retain a separate ten-minute limit.

## Download

- **Recommended:** `SpaceLens-Setup.exe` installs or upgrades SpaceLens and maintains Desktop/Start Menu shortcuts.
- **Portable:** `SpaceLens.exe` runs directly without installation.

This release is not represented as Authenticode-signed unless the files on the release page show a valid SignPath Foundation signature. Until SignPath onboarding is complete, Windows may show an unknown-publisher or SmartScreen warning. Download only from the official GitHub Releases page and verify the accompanying SHA-256 files.
