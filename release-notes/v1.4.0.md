# SpaceLens 1.4.0

SpaceLens 1.4 expands protected-location coverage while making dangerous files unmistakable before any cleanup action.

## Added

- **Full access scan (Administrator)** scans local fixed drives through a short-lived elevated scanner and an authenticated, current-user-only channel. The main interface stays unelevated, only Windows backup privilege is enabled, and ownership and permissions are never changed. Standard mode remains available for network folders.
- Real file rows now retain scan-time attributes and show a clear **Standard**, **Important**, or **Protected — do not delete** safety note.
- **Protected / important** is available as an independent category filter.
- Protected Windows/system files require an exact typed confirmation before SpaceLens attempts to move them to the Recycle Bin. Windows remains the final permission barrier.
- Cleanup now uses Windows Recycle Bin-only mode and refuses the operation when Windows cannot recycle the item; it never silently converts the action into permanent deletion.

## Fixed

- Access-denied files are no longer mistaken for files that disappeared after a scan.
- Recycling a file no longer makes its allocation appear as newly increased System / protected storage; the saved accounting is marked for a rescan instead.
- Multiple SpaceLens windows can no longer race the same saved scan or block an in-app update unexpectedly.
- Drive usage captured before and after a scan is retained so concurrent disk changes can be disclosed instead of presented as false precision.
- Saved-scan format v5 preserves safety and full-access metadata while remaining compatible with v4 and v3 caches.

## Important limits

Full access scan maximizes coverage of ordinary file entries that Windows exposes. NTFS metadata, restore points, reserved storage, alternate streams, and other filesystem-managed allocations are not necessarily ordinary files and cannot be guaranteed to appear as deletable rows or make System / protected reach zero.

Directory links and unknown directory reparse tags remain untraversed to prevent loops and paths escaping the selected root; recognized Microsoft Cloud Files directory tags remain supported. Full access elevation must use the same Windows account as SpaceLens.

SpaceLens still uses the Recycle Bin only. The elevated helper is scan-only, so an ACL-protected row can be visible but remain non-recyclable from the unelevated interface; the refusal is shown and the row is retained. SpaceLens never takes ownership, rewrites ACLs, stops services, schedules boot-time deletion, or silently falls back to permanent deletion.
